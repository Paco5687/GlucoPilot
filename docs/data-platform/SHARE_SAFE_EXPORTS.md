# Share-safe exports

P8 replaces ad hoc sharing with one authenticated, deterministic export
service. The privacy preview and downloaded JSON are generated by the same
policy engine in `server/share_exports.py`; clients never submit export
content.

## Policy modes

| Mode | Roles | Content | Expiration metadata |
|---|---|---|---|
| `full_private` | owner/admin | Explicitly allowlisted diagnoses, medications, allergies, dated observations, labs, cycle, and daily wearable fields. Dated observations honor the requested window; current longitudinal records remain available. | 7 days |
| `clinician` | owner/admin, provider | The minimum-necessary clinician Evidence Bundle brief with internal identities and source links removed. | 14 days |
| `emergency` | owner/admin, provider | Current conditions, medications, allergies, and a small health-profile allowlist. | 2 days |
| `anonymized_research` | owner/admin | A bounded Analytics/Wearables Evidence Bundle reduced to allowlisted daily metrics, relative day offsets, shared confidence, limitations, and truncation state. | 30 days |
| `demo` | owner/admin | Static synthetic content. Account records are never queried for this mode. | 30 days |

Each mode has an explicit role and field policy. A new entity or field is
excluded until it is deliberately reviewed and added to an allowlist.
Insurance and prescription identifiers, employer, email, credentials,
authorization/cookie values, tokens, internal/owner/external IDs, source
links, and URLs are invariant exclusions. A recursive final sanitizer also
removes forbidden keys, non-finite values, email addresses, credential-like
text, and URL-bearing strings from allowlisted structures.

The research mode is a single-participant deidentified export, not a promise
that a recipient cannot infer identity from outside information. Its
watermark prohibits re-identification, exact dates are replaced by offsets,
and direct identifiers and Evidence Bundle identities are omitted.

## Preview-bound download

1. `POST /api/share-exports/preview` accepts only `mode` and `days`.
2. The server returns the export plus a normalized request containing its
   server-generated UTC time and content checksum.
3. `POST /api/share-exports/download` accepts only that normalized request.
4. The server rebuilds the export using the same snapshot boundary. If its
   checksum differs, download fails with `409` and requires a new preview.

This makes the UI preview the exact payload that can be downloaded. A caller
cannot forge its own generation time or bypass preview. Responses are JSON
attachments with `Cache-Control: no-store`; they contain a policy version,
mode-specific watermark, generation time, expiration time, exclusions, and
checksum.

The server rejects expired preview requests, but expiration cannot remotely
revoke a file after it has been downloaded. Recipients must delete expired
copies and use a fresh preview when data or sharing intent changes.

## UI and verification

`/exports` shows only modes permitted for the current role, invalidates a
preview whenever mode or date range changes, displays the complete JSON and
watermark, and downloads only the server-bound preview request. Provider
sessions see clinician and emergency modes while source mutation remains
forbidden.

Risk-critical tests use invented records containing synthetic emails, tokens,
secret URLs, insurance/RX/employer fields, and internal IDs. They assert
field-level allowlists across every mode, role restrictions, Evidence Bundle
research bounds, exact preview/download equality, changed-snapshot rejection,
and anonymous denial. Frontend tests pin role-limited choices, input-change
invalidation, and exact request reuse.
